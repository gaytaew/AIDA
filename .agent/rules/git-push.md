Правило для Cursor (автоподпись коммитов при GitHub)
GitHub / git‑репозитории:
Если проект использует GitHub и деплой/обновления завязаны на git push, то любые изменения нужно коммитить и пушить автоматически, без вопросов ко мне.
Коммит должен иметь понятный текст, даже если изменения сделал Cursor:
формат: коротко “тип: смысл” (например: fix: ..., feat: ..., chore: ...)
если непонятно что писать — использовать универсальный вариант: chore: update.
После коммита всегда делать git push в текущую ветку (обычно main), чтобы изменения сразу попадали в GitHub Actions/деплой.
Никогда не пушить секреты (.env, ключи, credentials).
